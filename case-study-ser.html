Case Study: Vocal Sentiment Transformer

Speech Emotion Recognition using Wav2Vec2 Transformer

Project Overview

In human–computer interaction, recognizing emotions from raw speech remains a core challenge. Traditional CNN-based models struggle on real-world noisy audio, failing to capture contextual cues such as tone variation and emphasis. This project applies the transformer architecture Wav2Vec2, pre-trained on 53,000+ hours of audio, to fine-tune for Speech Emotion Recognition achieving 79% accuracy on real-world noisy speech data (RAVDESS + TESS).

79%

Validation Accuracy (Noisy Environment)

Problem Statement

Real-world SER systems fail under: background noise, multi-speaker dialogue, changing emotion intensity, or short utterances. The aim was to build a model capable of:

  • High accuracy on noisy & real speech
  • Minimal feature engineering
  • Efficient deployment for production apps

Model Architecture

Instead of using MFCC + CNN/LSTM manually extracted features, this project uses a feature-less pipeline powered by self-attention:

  • Base Model: Facebook Wav2Vec2 pre-trained checkpoint
  • Fine-tuned classification head added (7 emotions)
  • Training with label masking + gradient checkpointing

Datasets and Preprocessing

  • TESS (Toronto Emotional Speech Set)
  • RAVDESS Emotional Speech Dataset
  • Added noise augmentation (airport, restaurant, fan sound)
  • Resampled to 16kHz, 1 channel

Results

The transformer-based model handled contextual speech variations better than CNN/MLP baselines. The model achieved significant improvement in anger vs. neutral classifications—previously the most misclassified categories.

Deployment

  • API: FastAPI
  • Frontend: Streamlit UI
  • Containerization: Docker

What I Learned

  • Self-attention is superior to convolution for sequential audio
  • Noise augmentation is crucial before deployment
  • Model inference speed must be optimized for real-time apps
